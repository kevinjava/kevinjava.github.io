Android M移动网络关闭下彩信发送流程

发表于 2017-03-14 更新于 2019-09-04

前提条件

  • 要让系统支持移动网络关闭下发彩信，首先要打开framework/opt/telephony/resources/res/values/下的开关.

实现原理

  • Mms在发送彩信的时候，发现当前无移动网络时，则请求mms apn注网，注网成功后，发送彩信相关数据，完成后并且没有pending的彩信，则请求断开网络。
  • 在dual-sim的时候，需要check activePhoneId 和 requestPhoneId, 如果不一样，再要断开activePhoneId, 然后按requestPhoneId注网。

时序图

详解

framework/base 层，由mms调用ConnectivityManager的requestNetwork入口，到ConnectivityService，进行封装NetworkRequest注册网络，再到NetworkFactory处理NetworkRequest于opt/telephony通信。

  • ConnectivityManager.java

  • ConnectivityService.java

  • NetworkFactory.java

opt/telephony 根据request选择相应的APN，再由Dataconnection状态机确认进行状态的转化，再由DcTracker进行trySetupData, 再到DataConnection状态机，发ril消息到modem.

  • ApnContext.java

  • DcTracker.java

  • DataConnection.java

注网成功后，发出network available的notification, Mms获取到ConnectivityManager的NetworkCallback之后，
发送彩信的数据，完成后关闭网络跟注网流程类似，不过都是调用的release network相关的方法，之后走到applyNewState之后cleanUpConnection.
Qcom平台有Bug，是发送成功后，网络没有关掉，可以参照以下代码，来解决该问题。
在apn type为mms时，cleanup 置为true.

Video Call Hide me简介

发表于 2017-03-13 更新于 2019-09-04

Hide/Show Me本来应该是由AP测直接调用onSetPauseImage就Ok,但是由于Qcom M平台并没有full 实现onSetPauseImage，导致AP不得不做workaround来实现此功能，所以才有本文。

Hide Me是什么

Hide/Show me 功能为在Video call时候，关闭camera，给对方展示默认或者自定义图片。

Hide Me实现细节

关闭camera, 停止采集视频流。

Check hide me image 是否存在。

  • 检查3G/4G对应的h263/h264文件是否存在

  • h263文件应由176x144分辨率的图片转化而来， h264应该由480X640图片转化而来。分辨率不对，会导致Peer device绿屏。

  • h263/h264文件转化

    1. 使用ffmpeg库，交叉编译, 放入res/raw目录。

    2. load ffmpeg.

3. **compress image.**

确定本机和Peer device的网络类型(3G/4G)

确定peer device类型是根据onUpdatePeerDimensions回调的width/height来决定

根据网络类型选择对应H263或H264下发ims进行传输展示。

普通Video Call的时候，本机或者peer device有一个处在3G网络，则选择h263文件传输。只有双方都处于4G网络时才选择h264.
在conference call的时候并且是SKT运营商时，由本机网络决定h263还是h264, 3G->h263, 4G->h264.

Python中使用mysql数据库

ubuntu下安装mysql,python-mysqldb

sudo apt-get install mysql-client mysql-server
sudo apt-get install libmysqlclient-dev
sudo apt-get install python-mysqldb

根据提示输入密码即可。

使用python进行相关操作

导入包

import MySQLdb #注意大小写

创建数据库


    # encoding: utf-8
    #!/usr/bin/python

    import MySQLdb
    __author__ = 'kevliu'

    # 建立连接
    conn = MySQLdb.connect(host="localhost", user= "root", passwd= "123456")

    # 获取操作的游标
    cursor = conn.cursor()

    # 执行cursor语句创建数据库photo
    cursor.execute("""create database photo """)

    # 操作完毕，关闭游标
    cursor.close()

创建表


    # 选择photo db
    conn.select_db('photo')

    # 获取操作游标
    cursor = conn.cursor()

    # 执行插入表语句
    cursor.execute(""" create TABLE recodes(id varchar(100) UNIQUE , data VARCHAR(10000)) """)

    # 关闭游标
    cursor.close()

插入一条数据


    # dummy data
    value = ["test id", "test data"]

    cursor.execute("""insert INTO recodes VALUES (%s, %s)""", value)

    # 关闭游标
    cursor.close()

    # 提交数据
    conn.commit()

插入多条数据


    values = []
    for i in range(5):
        values.append(("test id" + str(i+1), "test data"+str(i+1)))
    # 使用executemany    
    cursor.executemany("""insert INTO recodes VALUES (%s, %s)""", values)

查询数据


    # 执行查询语句                                                       
    count = cursor.execute(""" select * from recodes """)          
    print str(count)                                               

    #获取游标中的第一条数据                                                   
    data = cursor.fetchone()                                       
    print 'ID: %s   info: %s' % data                               

    #获取游标中的5条数据                                                    
    datas = cursor.fetchmany(5)                                    
    for r in datas:                                                
        print r                                                    
    #重置游标位置，0,为偏移量，mode＝absolute | relative,默认为relative,           
    cursor.scroll(0,mode='absolute')                               
    #获取所有结果                                                        
    results = cursor.fetchall()                                    
    for r in results:                                              
        print r

解决mysql乱码问题

修改my.cnf配置文件 local: /etc/mysql/my.cnf

  1. 在[client]字段里加入default-character-set=utf8，如下：
    [client]
    port = 3306
    socket = /var/lib/mysql/mysql.sock
    default-character-set=utf8

  2. 在[mysqld]字段里加入character-set-server=utf8，如下：
    [mysqld]
    port = 3306
    socket = /var/lib/mysql/mysql.sock
    character-set-server=utf8

  3. 在[mysql]字段里加入default-character-set=utf8，如下：
    [mysql]
    no-auto-rehash
    default-character-set=utf8

  4. python连接是指定编码

    MySQLdb.connect(host=”localhost”, user= “root”, passwd= “123456”, charset=’utf8’, init_command=’SET NAMES UTF8’)

Android 远程调试

发表于 2015-08-13 更新于 2019-09-04

Android远程连接手机调试

首先在手机上安装终端模拟器

  • 搜索 Terminal Emulator for Android

打开手机上调试的端口，默认是5555

  • 打开终端，输入su，一定要在root下。

  • 输入 setprop service.adb.tcp.port 5555

  • 重启adbd进程，输入stop adbd & start adbd

电脑远程连接手机

  • 输入netcfg查看查看终端IP

  • 在电脑的终端／CMD里输入 adb connect IP
